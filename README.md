# Artifacto

Artifacto is an automated pipeline that transforms ephemeral AI agent conversations (bug fixes, architectural workarounds, novel solutions) into a permanent, structured, public knowledge base hosted on GitHub Pages.

## How it works

1. **Ingest**: Send raw text, a markdown file, or a chat URL to the Artifacto CLI.
2. **Refine**: The CLI uses an LLM (Claude, Ollama, etc.) to transform the raw context into a polished technical blog post with relevant tags and metadata.
3. **Review**: The post is saved as a local draft. You can preview, edit, or reject it.
4. **Publish**: Once approved, the post is moved to the site directory, committed, and pushed. GitHub Actions automatically builds and deploys the Astro-powered site to GitHub Pages.

## Architecture

- **CLI Tool (`artifacto`)**: The primary interface for ingesting and managing content.
- **Python Backend**: Handles ingestion (scraping, text extraction), LLM refinement, tag indexing, image processing, and git operations.
- **Astro Blog**: A highly optimized static site generated by Astro, featuring a dark developer-centric theme, client-side search (via Pagefind), and tag browsing.

## Setup

1. **Clone the repository**:
   ```bash
   git clone git@github.com:padeler/artifacto.git
   cd artifacto
   ```

2. **Install Python dependencies**:
   Ensure you have Python 3.11+.
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -e ".[dev]"
   ```

3. **Install Node.js dependencies**:
   ```bash
   cd site
   npm install
   cd ..
   ```

4. **Configure Environment Variables**:
   Copy `.env.example` to `.env` and fill in the required values.
   ```bash
   cp .env.example .env
   # Edit .env
   ```

## Usage

### Ingesting Content
```bash
# From a file
artifacto ingest --file ./my-notes.md

# From a URL
artifacto ingest --url "https://chat-log-link.com/123"

# From standard input
cat snippet.txt | artifacto ingest --pipe
```

### Reviewing Drafts
```bash
# List all pending drafts
artifacto review

# View a specific draft
artifacto review my-post-slug
```

### Publishing & Managing
```bash
# Approve a draft (commits and pushes to GitHub)
artifacto approve my-post-slug

# Reject a draft (deletes it)
artifacto reject my-post-slug

# List published posts
artifacto list

# Delete a published post
artifacto delete my-post-slug
```

## Local Development

To run the Astro site locally:
```bash
cd site
npm run dev
```

To run the Python tests:
```bash
pytest
```
